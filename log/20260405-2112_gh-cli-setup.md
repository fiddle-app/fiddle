# gh CLI Installation & Login (20260405-2112)

## What Was Done

Installed GitHub CLI (`gh`) via winget and authenticated both GitHub accounts.

## Installation

```
winget install GitHub.cli
```

## Accounts Configured

Two accounts authenticated in `gh`:

| Account | Purpose | SSH Key |
|---|---|---|
| `caseymullen` | Personal projects (e.g. travel) | `~/.ssh/id_ed25519_personal` |
| `fiddle-app` | Fiddle app family repos | `~/.ssh/id_ed25519_fiddle` |

Default active account: `caseymullen`

## SSH Key Titles on GitHub

When prompted for "Title for your SSH key" during `gh auth login`:
- Used a machine identifier (e.g. `casey-desktop` and `caseymullen-desktop`) rather than the account name,
  since the key is already scoped to that account's settings page.

## Switching Accounts

```bash
gh auth switch -u fiddle-app    # before fiddle repo work
gh auth switch -u caseymullen  # to return to default
```

## Notes

- `gh auth switch` is global — it applies to whatever `gh` command runs next, regardless of folder.
- Day-to-day git (push/pull/fetch) does NOT require switching — SSH handles account routing automatically.
- `gh` is needed for: repo creation, PR management, GitHub-specific actions.
